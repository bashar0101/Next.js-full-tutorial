Tutorial 2

Installing Next.js & Your First Project — From zero to a running app

📚 Level: Beginner ⏱ ~20 min read 🗓 May 2026 ✅ Exam Passed: 10/10
1 What Do You Need First? — Node.js

Before you can use Next.js, you need to install Node.js on your computer.

💡 JavaScript normally lives inside a browser (like Chrome). Node.js is like giving JavaScript a home on your computer so it can run anywhere — without a browser.

Next.js needs Node.js to run on your machine during development.

Check if you already have it:

$ node -v
v18.17.0 ← if you see this, you're good! ✅

If you get an error, go to nodejs.org and download the LTS version.

⚠️ Minimum required: Node.js version 18 or higher.
2 Creating Your First Next.js Project

Open your terminal and run this single command:

$ npx create-next-app@latest my-first-app
PartMeaning
npxRuns packages without installing them globally
create-next-appOfficial tool by Vercel to create Next.js projects
@latestUse the newest version
my-first-appYour project name (choose anything you like)

Answer the setup questions like this:

✔ Would you like to use TypeScript? → No
✔ Would you like to use ESLint? → Yes
✔ Would you like to use Tailwind CSS? → No
✔ Would you like to use the src/ directory? → No
✔ Would you like to use App Router? → No ← Important!
✔ Would you like to customize import alias? → No
⚠️ Why "App Router → No"? Next.js has two systems: the older Pages Router and the newer App Router. We start with Pages Router because it is easier to understand. We'll learn App Router later!
3 Opening the Project

Enter your project folder and open it in VS Code:

$ cd my-first-app
$ code .

If code . doesn't work, open VS Code manually and use File → Open Folder.

4 The Project Structure
my-first-app/
│
├── pages/ ← Your pages live here
│   ├── index.js ← Home page ( / )
│   ├── _app.js ← Wraps all pages
│   └── api/ ← API routes folder
│      └── hello.js ← Example API route
│
├── public/ ← Static files (images, icons)
│
├── styles/ ← CSS files
│   ├── globals.css ← Applied to all pages
│   └── Home.module.css ← Only for home page
│
├── node_modules/ ← Installed packages (don't touch!)
├── package.json ← Project info and dependencies
├── next.config.js ← Next.js configuration
└── .gitignore ← Files to ignore in Git

The most important folder is pages/ — this is where you spend most of your time. Every file here becomes a page automatically.

5 Running Your App

Start the development server with:

$ npm run dev

You will see:

▲ Next.js 14.x.x
- Local: http://localhost:3000
- Ready in 1.2s ✓

Open your browser and go to http://localhost:3000 — your app is running! 🎉

💡 What is localhost:3000? — localhost means "my own computer". 3000 is the port number (like a door number). So this means: "open the website running on my computer at door 3000".

To stop the server, press Ctrl + C in the terminal. To start again, run npm run dev.

6 Your First Page — pages/index.js

Open pages/index.js and replace everything with this simple code:

export default function Home() {
  return (
    <div>
      <h1>Hello, Next.js!</h1>
      <p>My first Next.js page is working!</p>
    </div>
  );
}

Save the file — the browser updates instantly! This is Fast Refresh.

💡 What is export default? — It means "this is the main thing from this file." In Next.js, it also tells the framework: "this is the component to render for this page." Without it, Next.js won't know what to display.
7 Creating a Second Page

Create a new file pages/about.js:

export default function About() {
  return (
    <div>
      <h1>About Page</h1>
      <p>This is the about page.</p>
    </div>
  );
}

Go to http://localhost:3000/about in your browser — your about page is live! No router config needed. 🎉

8 Navigating Between Pages — <Link>

Use the <Link> component from Next.js to navigate between pages:

import Link from 'next/link';

export default function Home() {
  return (
    <div>
      <h1>Hello, Next.js!</h1>
      <Link href="/about">Go to About Page</Link>
    </div>
  );
}
❌ Don't use — <a>

Reloads the full page every time. Slow. Normal HTML tag.

✅ Use this — <Link>

No page reload. Instant navigation. Great for performance.

The <Link> component uses client-side navigation — it only loads the new page's content, not the entire page again.

9 Summary
  • Install Node.js 18+ before using Next.js. Check with node -v.
  • Create a project with npx create-next-app@latest project-name.
  • Start the dev server with npm run dev → open http://localhost:3000.
  • The pages/ folder is where all your pages live — one file = one route.
  • Use export default to tell Next.js which component to render for the page.
  • Use <Link> (not <a>) to navigate — it's faster and avoids full page reloads.
  • Fast Refresh updates the browser instantly when you save a file.
