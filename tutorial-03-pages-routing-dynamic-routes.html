Pages, Routing & Dynamic Routes — Building flexible URL structures

📚 Level: Beginner → Intermediate ⏱ ~25 min read 🗓 May 2026 ✅ Exam Passed
1 Quick Review — Static Routes

Files in pages/ become pages automatically. These are static routes — the URL is fixed and never changes.

pages/index.js → /
pages/about.js → /about
pages/contact.js → /contact
pages/blog/news.js → /blog/news

Static routes work great for fixed pages. But for things like blog posts, products, or user profiles — you need dynamic routes.

2 Dynamic Routes — One File, Many URLs

Instead of creating 100 files for 100 blog posts, you create one file using square brackets [ ] in the filename:

pages/blog/[id].js ← one file handles ALL of these:

/blog/1
/blog/42
/blog/my-first-post
/blog/anything-at-all
💡 Think of [id] as a placeholder — like a blank space that gets filled in with whatever is in the URL.
3 The useRouter Hook — Reading URL Values

To read the value from the URL, use the useRouter hook from Next.js:

import { useRouter } from 'next/router';

export default function BlogPost() {
  const router = useRouter();
  const { id } = router.query; // reads the [id] from the URL

  return (
    <div>
      <h1>You are reading post: {id}</h1>
    </div>
  );
}
LineWhat it does
import { useRouter }Brings in the useRouter tool from Next.js
const router = useRouter()Creates a router object with all URL information
const { id } = router.queryExtracts the id value from the URL
{id} in JSXDisplays the value on the page
⚡ JavaScript Note — Destructuring

Destructuring is a shortcut to extract values from any JavaScript object. It is NOT specific to URLs — it works with any object!

// Without destructuring (longer way):
const id = router.query.id;

// With destructuring (shorter, same result):
const { id } = router.query;

// Works with any object:
const person = { name: "Bashar", age: 25 };
const { name, age } = person; // name = "Bashar", age = 25
4 Linking to Dynamic Routes

Use template literals to build dynamic URLs inside a <Link>:

import Link from 'next/link';

const posts = [1, 2, 3, 4, 5];

export default function Home() {
  return (
    <div>
      {posts.map((id) => (
        <div key={id}> // ← key is required in .map()!
          <Link href={`/blog/${id}`}>Read Post {id}</Link>
        </div>
      ))}
    </div>
  );
}
⚡ JavaScript Note — Template Literals

Use backticks ` ` with ${variable} to build strings with variables. Cleaner than string concatenation with +.

const id = 5;

// Old way:
const url = "/blog/" + id; // "/blog/5"

// Template literal (better):
const url = `/blog/${id}`; // "/blog/5" ← cleaner!
5 Nested Dynamic Routes

You can have multiple dynamic params in the same route:

pages/shop/[category]/[productId].js

→ /shop/shoes/42 category="shoes", productId="42"
→ /shop/shirts/7 category="shirts", productId="7"
→ /shop/books/99 category="books", productId="99"
import { useRouter } from 'next/router';

export default function Product() {
  const router = useRouter();
  const { category, productId } = router.query;

  return (
    <div>
      <p>Category: {category}</p>
      <p>Product ID: {productId}</p>
    </div>
  );
}
6 Query Parameters

Query parameters are the extra data after ? in a URL. router.query reads them too:

// URL: /search?term=nextjs

const { term } = router.query; // term = "nextjs"

Adding query params to a Link:

// Simple way:
<Link href="/search?term=nextjs">Search</Link>

// Object way (cleaner for multiple params):
<Link href={{ pathname: '/search', query: { term: 'nextjs' } }}>
  Search
</Link>
7 Programmatic Navigation — router.push()

Navigate using code — useful after form submissions or button clicks:

import { useRouter } from 'next/router';

export default function Home() {
  const router = useRouter();

  function goToAbout() {
    router.push('/about'); // navigate to /about
  }

  return (
    <button onClick={goToAbout}>Go to About</button>
  );
}
8 Custom 404 Page

Create pages/404.js to show a custom page when a URL doesn't match any route:

export default function NotFound() {
  return (
    <div>
      <h1>404 — Page Not Found</h1>
      <p>Sorry, this page does not exist!</p>
    </div>
  );
}
9 Summary — Routes Cheat Sheet
pages/index.js → /
pages/about.js → /about
pages/blog/[id].js → /blog/1 , /blog/42 , /blog/anything
pages/shop/[category]/[id].js → /shop/shoes/42
pages/404.js → shown when no route matches
  • Use [param].js filename syntax to create dynamic routes.
  • useRouter() gives you a router object with all URL information.
  • router.query holds both dynamic params and query string values.
  • Use destructuring to cleanly extract values from objects: const { id } = router.query.
  • Use template literals to build dynamic URLs: `/blog/${id}`.
  • Always add a key prop when rendering lists with .map().
  • router.push('/path') navigates programmatically using code.
